Esquema de Prompts para ChatGPT -> General

FORMATO TAO (TAREA - ACCIÓN - OBJETIVO)

  • Investigar y redactar un informe detallado sobre las tendencias actuales en la estrategia de marcas.
  • Examinar cómo las marcas están adaptando su posicionamiento y comunicación para responder a las expectativas cambiantes de los consumidores en un entorno digital y globalizado.
  • OBJETIVO: Proporcionar recomendaciones prácticas para fortalecer la identidad y la relevancia de las marcas en el mercado actual.

FORMATO RTF (ROL - TAREA - FORMATO)

  • ROL: Actúa como un experto en copywriting para una agencia de publicidad digital.
  • TAREA: Desarrollar una serie de copias persuasivas.
  • FORMATO: Motivador para promover una nueva campaña de lanzamiento de una aplicación móvil de fitness.

FORMATO ADP (ANTES - DESPUÉS - PUENTE)

  • ANTES: Una empresa de consultoría en marketing digital enfrenta una baja en la generación de leads en su sitio web.
  • DESPUÉS: Al revisar el diseño y aplicar una estrategia de contenido SEO, aumentan la tasa de generación de leads en un 30%.
  • PUENTE: ¿Qué más podríamos hacer para mejorar aún más la generación de leads y maximizar el impacto de nuestras campañas?

Esquema de Prompts para Desarrolladores Web

FORMATO TAO (TAREA - ACCIÓN - OBJETIVO)

Ejemplo 1

  • TAREA: Crear una página web básica con HTML y CSS.
  • ACCIÓN: Diseña una estructura HTML para una página de inicio que incluya un encabezado, una sección de contenido y un pie de página. Luego, aplica estilos CSS para que sea visualmente atractiva.
  • OBJETIVO: Practicar la estructura básica de HTML y el uso de CSS para estilizar elementos.

Ejemplo 2

  • TAREA: Resolver una actividad del profesor sobre JavaScript.
  • ACCIÓN: Escribe una función en JavaScript que tome un array de números y devuelva la suma de todos los elementos.
  • OBJETIVO: Entender cómo trabajar con arrays y funciones en JavaScript.

Ejemplo 3

  • TAREA: Comenzar un trabajo práctico con React.
  • ACCIÓN: Crea un componente React simple que muestre un mensaje de bienvenida en la pantalla.
  • OBJETIVO: Familiarizarse con la creación de componentes básicos en React.

FORMATO RTF (ROL - TAREA - FORMATO)

Ejemplo 1

  • ROL: Eres un estudiante de desarrollo web que está aprendiendo HTML y CSS.
  • TAREA: Diseña una página web que muestre una lista de tus películas favoritas.
  • FORMATO: Usa una lista desordenada (<ul>) en HTML y aplica estilos CSS para que cada elemento de la lista tenga un fondo de color y un borde redondeado.

Ejemplo 2

  • ROL: Eres un estudiante que está resolviendo una duda sobre JavaScript.
  • TAREA: Explica cómo funciona el método map() en JavaScript y proporciona un ejemplo práctico.
  • FORMATO: Escribe una explicación clara y un ejemplo de código que transforme un array de números en un array de sus cuadrados.

Ejemplo 3

  • ROL: Eres un estudiante que está empezando con React.
  • TAREA: Crea un componente React que muestre un botón y, al hacer clic en él, muestre un mensaje en la consola.
  • FORMATO: Proporciona el código completo del componente y explica cómo funciona el manejo de eventos en React.

FORMATO ADP (ANTES - DESPUÉS - PUENTE)

Ejemplo 1

  • ANTES: Un estudiante está comenzando un proyecto web y no sabe cómo estructurar su archivo HTML.
  • DESPUÉS: Después de aprender sobre las etiquetas básicas de HTML, crea una estructura correcta con <header>, <main> y <footer>.
  • PUENTE: ¿Qué más puedo hacer para mejorar la semántica de mi HTML y hacer que mi página sea más accesible?

Ejemplo 2

  • ANTES: Un estudiante tiene una duda sobre cómo centrar un div en CSS.
  • DESPUÉS: Después de investigar, aprende a usar display: flex y justify-content: center para centrar el div.
  • PUENTE: ¿Cómo puedo aplicar estos conceptos para centrar elementos vertical y horizontalmente en una página?

Ejemplo 3

  • ANTES: Un estudiante está empezando con React y no sabe cómo crear un componente funcional.
  • DESPUÉS: Después de seguir un tutorial, crea un componente funcional que muestra un mensaje en la pantalla.
  • PUENTE: ¿Cómo puedo pasar datos a este componente usando props y hacerlo más dinámico?
